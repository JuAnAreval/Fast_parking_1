Informe de análisis de la aplicación
===============================

Fecha: 29 de abril de 2026
Proyecto: parqueaderos

1. Estructura general del repositorio
-------------------------------------
- backend/: API REST en Node.js.
- web/: aplicación web frontend con React y Vite.
- mobile/: aplicación móvil en Flutter.
- env, parqueaderos_db.sql, lib/: recursos y datos adicionales.

2. Tecnologías usadas
---------------------

2.1 Backend
-----------
- Lenguaje: JavaScript (Node.js).
- Framework HTTP: Express 5.
- Base de datos: MySQL (cliente mysql2).
- Seguridad y autenticación:
  - jsonwebtoken para tokens JWT.
  - bcryptjs para hash de contraseñas.
- Otros paquetes:
  - dotenv para variables de entorno.
  - cors para habilitar CORS.
  - nodemailer para envío de correos.
- Tests:
  - jest.
  - supertest.
  - cross-env para configuración de entorno en tests.

2.2 Web frontend
----------------
- Lenguaje: JavaScript/JSX.
- Framework/biblioteca: React 19.
- Bundler/desarrollo: Vite.
- Estilos: Tailwind CSS.
- Ruteo: React Router DOM.
- HTTP: Axios.
- Mapas/geo: Leaflet y react-leaflet.
- Entorno: servidor Node.js local para inicio/preview.

2.3 Mobile frontend
-------------------
- Lenguaje: Dart.
- Framework: Flutter.
- SDK objetivo: Flutter compatible con Dart SDK ^3.9.2.
- Paquetes principales:
  - flutter_map para mapas.
  - latlong2 para manejo de coordenadas.
  - geolocator para ubicación.
  - shared_preferences para almacenamiento local.
  - url_launcher para abrir enlaces.
  - cupertino_icons.

3. Arquitectura
---------------

3.1 Backend
-----------
- La aplicación backend usa un patrón de tipo MVC / arquitectura en capas.
- Se observa claramente la separación entre:
  - routes/: define las rutas de API y enlaza con los controladores.
  - controllers/: implementa la lógica para cada endpoint.
  - models/: encapsula el acceso a datos SQL con consultas específicas.
  - services/: lógica de negocio adicional como verificación de email y recuperación de contraseña.
  - middlewares/ y middleware/: manejo de autenticación y autorización.
  - config/: configuración de conexión DB y esquema.
- Ejemplo:
  - `backend/routes/parqueaderoRoutes.js` expone rutas REST.
  - `backend/controllers/parqueaderoController.js` maneja las peticiones.
  - `backend/models/usuarioModel.js` interactúa con la base de datos.
- Esto confirma un enfoque MVC o similar basado en capas de rutas/controladores/modelos.

3.2 Frontend y mobile
---------------------
- La app web es una SPA React que consume la API del backend.
- La app móvil Flutter también parece ser un cliente separado que consume servicios similares.
- No hay evidencia de un patrón MVC estricto en el frontend, pero sí un diseño cliente-servidor.

4. Conclusión
-------------
- El backend está construido con Node.js + Express + MySQL y sigue una arquitectura MVC/lámina de rutas-controladores-modelos.
- El frontend web es React + Vite + Tailwind + Leaflet.
- La aplicación móvil es Flutter + Dart con librerías de mapas y localización.
- En conjunto, el proyecto es una solución multiplataforma con API REST compartida entre web y móvil.
